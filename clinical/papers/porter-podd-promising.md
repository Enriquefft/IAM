# Question
Does Pragmatic Organisation Dynamic Display (PODD) communication books represent a promising practice for individuals with ASD?

# Method
- Position / framework paper.
- Authors include PODD developer (Gayle Porter).
- Reviews PODD design rationale and case-level reports.

# Findings
- PODD is a paper-based or app-based system organized by pragmatic communication functions ("I'm telling you something", "I want", "Let's pretend", "I need help").
- Built on aided language input theory: speaking partner models on PODD before expecting AAC user to use it.
- Designed for motor automaticity — vocabulary lives in consistent locations across pages.
- Marketed for "all-the-time use" (carried by user constantly).
- Limited RCT evidence base; mostly case-level reports and clinical experience.

# Builder takeaways
- PODD's pragmatic-branch organization is a UX pattern worth borrowing: organize by *communicative intent*, not by alphabet or category.
- Aided language input ("modeling on the AAC system") is consistent with [barker2013-aac-preschool-language] findings on peer modeling.
- "All-the-time use" implies tablet-based AAC must work offline, in low-light, with audio off, etc. — survey real environments before designing.
